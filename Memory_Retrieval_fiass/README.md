# Four-Role Cognitive Agent with FAISS Knowledge Base

A cognitive agent built on a four-role architecture (Memory · Perception · Decision · Action) with a FAISS-backed vector knowledge base, Ollama-powered embeddings, and a web UI for document indexing and Q&A.

---

## Architecture

```
agent6.py              ← main agent loop (12 iterations max)
  ├── memory.py        ← FAISS vector search + keyword fallback, persists to state/
  ├── perception.py    ← goal decomposition + completion tracking (Gemini-pinned)
  ├── decision.py      ← answer or tool call for one goal at a time (Gemini)
  ├── action.py        ← MCP tool dispatch, stores results in state/artifacts/
  ├── artifacts.py     ← artifact store (art:1, art:2, …)
  ├── gateway.py       ← LLM gateway client (Ollama embed + LLM Gateway V3 for chat)
  ├── schemas.py       ← all Pydantic v2 contracts
  ├── vector_index.py  ← FAISS IndexFlatIP wrapper with disk persistence
  └── mcp_server.py    ← 12 MCP tools: web_search, fetch_url, read_file,
                          index_document, index_paper_md, search_knowledge,
                          memory_stats, create_file, update_file, …

ui_app.py              ← FastAPI web UI (port 8200)

extract_url_texts.py   ← fetches URLs from .md paper files → clean .txt extracts
index_via_faiss.py     ← indexes .txt files into FAISS via mcp_server
```

### Knowledge Base Pipeline

```
sandbox/papers/*.md          (paper reference files with URLs)
        ↓
extract_url_texts.py          (fetches + cleans page content via BeautifulSoup)
        ↓
sandbox/papers/url_text_exports/*.txt   (one txt per URL, abstract-only variant included)
        ↓
index_via_faiss.py            (chunks → Ollama nomic-embed-text → FAISS + memory.json)
        ↓
state/index.faiss + state/memory.json   (persistent vector index)
        ↓
agent6.py / ui_app.py         (search_knowledge → synthesise answer)
```

## Prerequisites

| Requirement | Details |
|-------------|---------|
| Python | 3.11+ |
| uv | Package manager |
| Ollama | Running at `http://localhost:11434` |
| nomic-embed-text | `ollama pull nomic-embed-text` |
| qwen3.5 (or any model) | `ollama pull qwen3.5` — used by UI for answer generation |
| LLM Gateway V3 | Running at `http://localhost:8101` — used by agent6.py for Perception/Decision |

> **Embeddings** use Ollama directly (no gateway needed).
> **Chat/Structured output** for the agent roles routes through LLM Gateway V3.

## Indexing Papers

### Step 1 — Extract clean text from paper URLs

Place paper `.md` files in `sandbox/papers/`. Each `.md` file contains URLs to the paper (arxiv, doi, etc.). Then run:

```bash
python3.11 extract_url_texts.py
```

This fetches each URL, strips navigation/scripts via BeautifulSoup, and writes clean `.txt` files to `sandbox/papers/url_text_exports/`. For arxiv abs pages it also writes a dedicated `*__abstract.txt` with just the title and abstract for higher-quality embeddings.

**Notes:**
- "No HTML available" pages (arxiv papers without HTML) are automatically skipped; the abs URL is used as fallback
- Near-identical pages are deduplicated by content hash

### Step 2 — Index into FAISS

```bash
# Clear any stale index first
rm -f state/memory.json state/index.faiss state/index_ids.json

# Index all txt files from url_text_exports/
python3.11 index_via_faiss.py
```

Each file is chunked (400 words, 80-word overlap), embedded with `nomic-embed-text`, and stored in FAISS + `state/memory.json`.

## Web UI — Document Q&A

```bash
python3.11 ui_app.py
# Open http://localhost:8200
```

**Features:**
- **File browser** — shows all `.txt`/`.md` files from `extra_docs/` folder with checkboxes
- **Bulk select** — Select All / Deselect All
- **One-click indexing** — selected files are indexed into FAISS immediately
- **Indexed documents list** — shows all currently indexed documents with chunk counts
- **Query panel** — type a question, get an answer generated by `qwen3.5` from relevant chunks
- **Source attribution** — each answer shows source files with similarity scores (green ≥0.55, yellow ≥0.45, red <0.45)
- **Low-confidence warning** — if no relevant content found (score <0.50), warns instead of hallucinating

**To add your own documents:**
1. Copy `.txt` or `.md` files into `extra_docs/`
2. Refresh the browser — files appear in the checklist
3. Select and click **Index Selected**

## Running the Agent

```bash
python3.11 agent6.py "Your question here"
```

Documents indexed via the UI or `index_via_faiss.py` are immediately available to the agent — both share the same `state/` directory.

### Example queries

```bash
# Query indexed papers
python3.11 agent6.py "What are the key contributions of the Transformer architecture?"

# Cross-paper synthesis
python3.11 agent6.py "Across these papers, how do they handle training efficiency?"

# Chain-of-thought specific
python3.11 agent6.py "What does the chain-of-thought paper say about improving reasoning?"

# Memory stats
python3.11 agent6.py "How many chunks were indexed in total?"
```

## Reset State

```bash
rm -f state/memory.json state/index.faiss state/index_ids.json
```

This wipes the vector index and all memory. Re-run `index_via_faiss.py` to rebuild.

## Known Limitations

| Issue | Detail |
|-------|--------|
| CoT paper | No arxiv HTML version — only abs page indexed; abstract-only chunks improve retrieval |
| Off-topic queries | FAISS returns nearest neighbour even when unrelated; similarity score <0.50 triggers LOW CONFIDENCE warning |
| UI answer generation | `qwen3.5` can be slow; use `agent6.py` in terminal for faster structured retrieval |
| Gateway required | `agent6.py` Perception/Decision roles need LLM Gateway V3 at port 8101 |
| Duplicate chunks | Running `index_via_faiss.py` multiple times without clearing state accumulates duplicates — always clear state first |
